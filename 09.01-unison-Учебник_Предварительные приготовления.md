Учебник_Предварительные приготовления
Unison можно использовать с любым из двух пользовательских интерфейсов:
    1. текстовым интерфейсом и
    2. графическим интерфейсом

   Текстовый интерфейс более удобен для запуска из скриптов и
работает на немых терминалах; графический интерфейс лучше подходит для большинства
интерактивных применений. Для этого урока вы можете использовать любой из них. Если вы
запускаете Unison из командной строки, просто набрав unison, вы выберете
   либо текстовый, либо графический интерфейс, в зависимости от того, какой из них был создан.
   выбран по умолчанию при создании исполняемого файла, который вы запускаете. Вы
   можно принудительно использовать текстовый интерфейс, даже если по умолчанию используется графический, добавив
   -ui text. Другие аргументы командной строки для обеих версий
идентичны.

   Графическую версию также можно запустить напрямую, щелкнув по ее значку.
   В этом руководстве мы предполагаем, что вы запускаете ее с помощью команды
   линия.

   Unison может синхронизировать файлы и каталоги на одном компьютере или
между двумя компьютерами в сети. (На обоих компьютерах работает одна и та же программа
, разница лишь в том, какая из них отвечает за
отображение пользовательского интерфейса.) Если вас интересует только
   настройка выполняется на одной машине, тогда давайте назовем эту машину КЛИЕНТОМ. Если
вы синхронизируете две машины, давайте назовем их КЛИЕНТОМ и СЕРВЕРОМ.
